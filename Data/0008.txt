제가 쓰고 싶은데 아직 못 쓰고 있는 C++ 11의 기능이 하나 있어요 
constexpr(Const Expression)이라고 일반적으로 코드를 작성해서 하는 계산 같은 건데, 컴파일 도중에 코드를 다 실행을 해서 그 결과 값을 알 수가 있기 때문에 컴파일된 바이너리(binary) 최종 결과 값 하나만 넣을 수 있는 거예요. 
아주 간단한 예를 들어드리면 
i=1 
j=1 
a = i + j 
라는 코드가 있고 실제 코드에서는 a만 쓰고 싶어요. 그러면은 exe 파일에 계산하는 코드를 넣을 필요 없이 3줄을 합쳐서 a = 2라는 거 2만 구해 갖고, 그 2를 대신에 a가 쓰이는 곳에 모두 넣어주는 그런 개념이거든요. 

이런 게 지금까지 불가능한 건 아니었어요. 예를 들어서, 전처리 문(preprocessor)라고 있죠? #define 같은 거... 그런 거 써서 만드는 방법도 있었고, 컴파일러 최적화 flag 같은 걸로 올려두면 그걸로도 웬만해선 컴파일러가 최적화하는 경우도 있었어요. 그래서 어떤 문자열(string)에서 해쉬(hash) 값을 찾아내고 싶은데 이걸 컴파일 중에 찾았으면 좋겠다는 생각을 했어요. 왜냐하면 실행 시에는 문자열 하나씩 둘러보면서 일일이 계산을 해야 되니까... 아무래도 조금은 더 느릴 것 같아서 성능을 개선해보자란 생각으로 컴파일 시간에 문자열 해쉬를 만들어 보자라는 그런 제목으로 블로그를 올린 적이 있어요. (https://blog.popekim.com/ko/2012/01/11/compile-time-hash-string-generation.html) 
이런 걸 constexpr을 사용하면 다 해결할 수가 있죠. 문자열 해쉬 계산하는 것도 어느 정도 제약이 있고 컴파일러마다 최적화 정도도 조금씩 다르고요... 그래서 이리저리 사용해 본 결과 아직 그렇게 만만하게 쓸 만하진 않다는 생각을 했었거든요.

결과적으로 지금 constexpr을 못 쓰는 이유는 사실 하나예요. 근데 저는 누누이 얘기했듯이 비주얼 스튜디오를 사용하는 개발자고요, 아직 비주얼 스튜디오 2012에서 지원을 안 해서 못쓰고 있는 거예요. 
제가 알기로는 GCC나 Clang은 지원을 지금 하는 걸로 아는데, 아무래도 IDE가 좋지 않으면 프로그래밍하는 게 너무 힘들더라고요. 
분명히 마이크로소프트사가 분명히 이걸 빨리 만들어 달란 요청을 되게 많이 받고 있을 거예요. 근데 아직 만들었단 얘기를 제가 못 들었고 그래서 못 쓰고 있죠. 
지금 제가 알고 있는 C++ 11 기능 중에 정말 쓰고 싶은데 못 쓰고 있는 기능은 현재로서는 constexpr 같아요. 
그래서 혹시라도 이걸 보고 있는 마이크로소프트 직원들이 있으면 얼른 만들어 주시면 좋은데 제가 어차피 MS MVP기 때문에 물어볼 순 있을 것 같아요. 빨리 지원 쫌 해달라고... 
다른 중요한 것들이랑 만들고 있겠죠 예를 들어서 intelliSense를 더 좋게 만든다면은 constexpr 정도는 없어도 일단 참아줄 수 있을 것 같아요.
